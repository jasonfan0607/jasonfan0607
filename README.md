## 🤖 Agent Research Notes

- **Agent Runtime**：从零实现可运行的 Agent 主循环，关注 `LLM Action → Tool Call → Observation → Final Answer` 的闭环，而不是直接依赖现成框架。
- **Multi-Agent Collaboration**：探索 Router、Scheduler、Orchestrator、Adapter、消息协议与任务卡片，让多个 Agent 能像团队一样分工、协作和汇总。
- **Memory & State**：关注 session 持久化、结构化任务记忆、Trace 日志、上下文裁剪与跨轮次继续执行。
- **RAG / HIL / Evaluator**：尝试把知识检索、人机协作审批、质量评估与反馈重跑放进 Agent 工作流，提升可靠性。
- **Backend Reliability**：长期使用 Java / Spring Boot / Redis / MySQL 构建后端系统，习惯用工程约束兜底异常路径。

## 🧠 What I'm Building Toward

```text
User Intent
   ↓
Planner / Orchestrator
   ↓
Tool Calling + Memory + State Machine
   ↓
Traceable Execution
   ↓
Human-in-the-loop Review
   ↓
Reliable Agent Product
```

我希望构建的 Agent 不是一次性的玩具 Demo，而是一个具备以下特征的工程系统：

- **可运行**：有清晰主循环、工具协议、异常处理和最大步数限制。
- **可观测**：每一步 LLM 决策、工具参数、工具结果和错误都能被 Trace。
- **可协作**：不同 Agent 可以通过消息协议、任务状态和 Adapter 组合在一起。
- **可恢复**：session、tasks、artifacts、traces 能持久化，支持跨轮次继续执行。
- **可评估**：通过 Evaluator、反馈重跑和人工审批降低生成结果的不确定性。
